    路线图¶

    AgentMesh 从想法到可复用工具的演进路线。

    Phase 1 — 可行性研究（2026-05-18 ✅）¶

    目标：判断"跨主体Agent协作"是否可行

    任务 产出
    竞品深度扫描 9个竞品分析，定位差异化：保真度追踪+贡献度量化
    技术可行性 四层架构+七阶段计划
    经济模型 线性分配+Shapley值近似
    法律风险 签名溯源+人类在环+贡献度不涉及法律权属

    Phase 2 — 最小实验验证（2026-05-18 ✅）¶

    目标：证明Agent协作确实存在信息损失

    • exp-001：同一Agent模拟多角色 → 保真度0.539
    • exp-002：独立Agent → 保真度0.414，发现"丢信息"真实存在
    • 信任冷启动：质押担保原型
    • 经济模型：线性分配跑通

    Phase 3 — 协议原型（2026-05-19 ✅）¶

    目标：定义协作协议，验证Schema能减少冲突

    • Schema v2：5种消息类型+L1模板
    • exp-003：盲测 → 冲突率100%
    • exp-004：跨平台 → ACP权限兼容失败
    • 兼容层：自然语言→数值自动映射

    Phase 4 — SDK与保真度（2026-05-19 ✅）¶

    目标：集成SDK，验证保真度追踪和贡献度分配

    • SDK v0.1：4个模块原型
    • exp-005：v2.1盲测 → 冲突率100%→0%
    • 保真度追踪：累积衰减计算
    • 贡献度分配：Shapley值近似

    Phase 5 — MVP实现（2026-05-19 ✅）¶

    目标：从脚本到可运行的Demo

    • 场景定义：研究协作（A检索+B整合+C审查）
    • SDK集成版：4个模块统一为一个SDK
    • 签名模块：ECDSA/Ed25519可选
    • Demo可运行

    Phase 6 — 产品验证（2026-05-20 ✅）¶

    目标：系统验证L1模板的实际效果

    • exp-005：对照组3处遗漏 vs 实验组0处
    • exp-006：MVP场景跑通，0%冲突率
    • 四类问题四机制框架定稿

    Phase 7 — 技能打包（2026-05-20 ✅）¶

    目标：从实验工具→可复用软件

    • agentmesh-protocol技能上架
    • SDK v0.2集成
    • exp-008：审查Agent检测率100%
    • exp-009：4步恢复效应

    Phase 8 — 推广MVP（2026-05-20 ~ 2026-05-22 ✅）¶

    目标：从可复用→可发布

    • A2A适配器 v1.0（正向/反向转换+AgentCard）
    • CLI工具（5个命令）
    • 3个端到端示例
    • 开发者pitch+推广策略
    • GitHub仓库上线

    Phase 9 — 发布与生态对接（进行中）¶

    目标：从本地验证→真实生态

    • 本地发布平台（GitHub Pages + MkDocs）
    • 真实A2A SDK对接验证
    • GitHub CI/CD
    • ClawHub发布（等待授权）

    Phase 10 — 社区推广（规划中）¶

    目标：开发者社区建设

    • 技术博客+演示视频
    • 社区贡献指南
